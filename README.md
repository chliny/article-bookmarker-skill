# Article Bookmarker

[中文](README_zh-CN.md)

A powerful skill for collecting, organizing, and managing article bookmarks with AI-powered summaries and auto-tagging.

## Features

- **Smart Content Extraction**: Automatically fetch and extract article content from URLs using web scraping
- **AI-Powered Summarization**: Generate concise, informative summaries of articles using AI models
- **Auto-Tagging**: Automatically categorize articles with relevant tags based on content analysis
- **Searchable Collection**: Maintain a well-organized, searchable bookmark collection
- **Tag-Based Indexing**: Bidirectional mapping between tags and articles for easy navigation
- **Markdown Format**: All bookmarks stored in clean, portable markdown files

## Quick Start

### Basic Usage

#### Bookmark an Article from URL

```
User: "Bookmark this article: https://example.com/ai-research"
```

The skill will:
1. Fetch and extract the article content
2. Generate an AI summary
3. Auto-tag based on content analysis
4. Save to your bookmark collection
5. Update the tag index

#### Bookmark Text Content

```
User: "Bookmark this content: [paste your article text here]"
```

#### Delete a Bookmark

```
User: "Delete the article ai-research-2024.md"
```

The skill will show confirmation details before deletion.

## Configuration

### Bookmark Storage Location

Bookmarks are stored in a directory specified by the `ARTICLE_BOOKMARK_DIR` environment variable. You must set this variable before using the skill:

```bash
export ARTICLE_BOOKMARK_DIR="/path/to/your/bookmarks"
```

## File Structure

Bookmarks are stored in the configured directory:

```
bookmarks/
├── article-title-slug.md      # Individual bookmark files
├── TAG_INDEX.md               # Tag to article mapping
└── README.md                  # Directory overview
```

### Bookmark File Format

Each bookmark contains:

```markdown
# Article Title

**Source:** URL  
**Bookmarked:** YYYY-MM-DD HH:MM GMT+8  
**Tags:** tag1, tag2, tag3

## Summary

AI-generated concise summary of the article...

## Content

Full extracted article content...

## Original URL

[Link](URL)
```

## Tag Management

### Auto-Tagging Logic

Tags are generated by analyzing:
- Article domain and topic keywords
- Technical terms and concepts
- Content categories (tutorial, news, research, etc.)
- Named entities and proper nouns

### Tag Index

The `TAG_INDEX.md` file maintains:
- **Tags Section**: Lists all tags with links to associated articles
- **Articles by Tag Count**: Groups articles by the number of tags they have

## Implementation Details

### Content Extraction
- Uses `web_fetch` with markdown extraction mode
- Handles content truncation gracefully
- Preserves original formatting where possible

### Summary Generation
AI-generated summaries capture:
- Main thesis or argument
- Key insights or findings
- Practical implications or applications

Typical length: 150-300 words

### File Naming
- Converts title to lowercase
- Replaces spaces/special characters with hyphens
- Limits to ~50 characters
- Ensures uniqueness with numbered suffixes if needed

## Error Handling

- **Duplicate Files**: Automatically appends numbered suffixes
- **Network Failures**: Retries fetch operations with user notification
- **Invalid URLs**: Validates before fetching and requests corrections
- **Empty Content**: Warns user and prevents bookmark creation
- **Deletion Safety**: Always confirms before removing files

## Best Practices

1. Use descriptive, sanitized filenames
2. Balance tag specificity vs generality
3. Periodically review and clean unused tags
4. Backup critical bookmarks externally
5. Always confirm destructive actions

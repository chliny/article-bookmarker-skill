# Article Bookmarker

[English](README.md)

用于收集、组织和管理文章书签，支持 AI 智能摘要和自动标签的技能。

## 特性

- **智能内容提取**：使用网页抓取技术自动获取和提取 URL 中的文章内容
- **AI 驱动摘要**：使用 AI 模型生成简洁、有信息量的文章摘要
- **自动标签**：基于内容分析自动为文章打上相关标签
- **可搜索集合**：维护一个组织良好、可搜索的书签集合
- **标签索引**：标签与文章之间的双向映射，便于导航
- **Markdown 格式**：所有书签以整洁、可移植的 markdown 文件存储
- **GitHub 同步**：可选的 GitHub 仓库同步功能，实现备份和跨设备访问

## 快速开始

### 基本用法

#### 从 URL 收藏书签

```
用户: "帮我收藏这篇文章：https://example.com/ai-research"
```

技能将会：
1. 运行 `scripts/bookmark.sh init` 初始化书签目录
2. 获取并提取文章内容
3. 生成 AI 摘要
4. 基于内容分析自动打标签
5. 保存到您的书签集合
6. 更新标签索引
7. 运行 `scripts/bookmark.sh save` 提交并推送到 GitHub（如已配置）

#### 收藏文本内容

```
用户: "收藏这段内容：[粘贴您的文章文本]"
```

#### 删除书签

```
用户: "删除文章 ai-research-2024.md"
```

技能将会：
1. 运行 `scripts/bookmark.sh init` 确保目录就绪
2. 查找并显示文章详情
3. 删除前请求确认
4. 删除书签文件并更新标签索引
5. 运行 `scripts/bookmark.sh save` 提交并推送更改

## 配置

### 书签存储位置

书签存储在由 `ARTICLE_BOOKMARK_DIR` 环境变量指定的目录中。在使用此技能前，必须设置此环境变量：

```bash
export ARTICLE_BOOKMARK_DIR="/path/to/your/bookmarks"
```

### GitHub 同步（可选）

要将书签同步到 GitHub 仓库，请设置 `ARTICLE_BOOKMARK_GITHUB` 环境变量：

```bash
export ARTICLE_BOOKMARK_GITHUB="username/repo"
```

支持的格式：
- `username/repo`
- `git@github.com:username/repo.git`
- `https://github.com/username/repo`
- `https://github.com/username/repo.git`

配置后，书签将自动提交并推送到远程仓库。如果仓库不存在，将自动创建（默认为私有仓库）。

### 外部依赖

| 工具 | 必需 | 用途 |
|------|----------|---------|
| `git` | **是** | 书签管理的版本控制 |
| `gh` | 否 | 自动创建 GitHub 仓库，检查远程仓库是否存在 |

## 文件结构

书签存储在配置的目录中：

```
bookmarks/
├── article-title-slug.md      # 单个书签文件
├── TAG_INDEX.md               # 标签到文章的映射
└── README.md                  # 目录概览
```

### 书签文件格式

每个书签包含：

```markdown
# 文章标题

**来源：** URL  
**收藏时间：** YYYY-MM-DD HH:MM GMT+8  
**标签：** 标签1, 标签2, 标签3

## 摘要

AI 生成的文章简洁摘要...

## 内容

完整的提取文章内容...

## 原始链接

[链接](URL)
```

## 标签管理

### 自动标签逻辑

标签通过分析以下内容生成：
- 文章领域和主题关键词
- 技术术语和概念
- 内容分类（教程、新闻、研究等）
- 命名实体和专有名词

### 标签索引

`TAG_INDEX.md` 文件维护：
- **标签部分**：列出所有标签及其关联文章的链接
- **按标签数量分组的文章**：根据标签数量对文章进行分组

## 实现细节

### 内容提取
- 使用 `web_fetch` 进行 markdown 提取
- 优雅处理内容截断
- 尽可能保留原始格式

### 摘要生成
AI 生成的摘要包含：
- 主要论点或观点
- 关键见解或发现
- 实际意义或应用

典型长度：150-300 字

### 文件命名
- 将标题转换为小写
- 用连字符替换空格和特殊字符
- 限制在约 50 个字符
- 如需要，通过数字后缀确保唯一性

## 错误处理

- **重复文件**：自动附加数字后缀
- **网络故障**：重试获取操作并通知用户
- **无效 URL**：在获取前验证并请求更正
- **空内容**：警告用户并阻止书签创建
- **删除安全**：删除前始终确认

## 最佳实践

1. 使用描述性的、经过清理的文件名
2. 平衡标签的特异性和通用性
3. 定期审查和清理未使用的标签
4. 在外部备份重要书签
5. 始终确认破坏性操作
